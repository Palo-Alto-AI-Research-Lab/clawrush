## The claim (as recorded, de-dramatized)

- Old path: learn tool configuration from third-party YouTube guides; copy and repeat their setups.
- Observed shift: the most useful configuration knowledge now comes from interrogating the tool itself ("what did you do / what helped / why did you err"), because it answers about YOUR project, not the guide author's.
- Mechanism: each answer becomes a persistent rule the tool re-reads at session start, so setup compounds over time - a self-improving loop rather than a one-off config.

## Working rule adopted

- Apply only settings you understand and can repair yourself (AK-47 / simplicity gate); ask the tool to explain a setting before adopting it. Complex, brittle schemes are declined by default.
